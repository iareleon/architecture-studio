# Focus Guardian — Weekly Review Subflow

Run once per week. 15 minutes maximum.
Honest assessment of the week against the plan.

## 1. Read the week

Read `arch-agency/context/sequencing.md`.
Read `arch-agency/STATUS.md`.
Check social-media vault — posts published this week.
Check ClickUp — tasks completed (if FM-F-020 active).
Check `Obsidian/meta/inbox/` (and recent wiki / classify activity per your Knowledge OS log) — captures this week.

## 2. Score the week

Report against the three tracks:

```
Week of {date} — Review
══════════════════════════════════════════════

Track A — Soul & Vision
  Brain dumps completed:  {N}
  Wiki pages advanced:    {N}
  vision.md status:       {🟡/🟠/🟢}

Track B — Social Media
  Posts published:        {N} (target: 3)
  Posts drafted:          {N}
  Content signals acted:  {N}

Track C — Foundation
  Skills built:           {N}
  Vault audit progress:   {status}
  FM-F-020 status:        {done/pending}

Track D — Meridian Build
  Status:                 {gated / active}
  Sprints completed:      {N}

Drift events this week:   {N}
Soul captures:            {N}
Focus checks run:         {N}
```

## 3. Honest assessment

One sentence for each track — what happened vs what was planned.
No softening. No positivity bias. Just what is true.

## 4. One priority for next week

Ask:
> "Based on this week, what is the single most important thing to move next week?"

Listen. Agree or surface a conflict with the sequencing plan.
Update `references/current-tracks.md` if the priority changes.

## 5. Patterns

After 3+ weekly reviews, check for recurring patterns:
- Always drifting to the same activity → may need to be scheduled, not avoided
- Always missing social media → system friction, not motivation problem
- Soul layer empty for weeks → governing intelligence not being fed
- SkillsLoom skills queued but not built → capability debt accumulating

Name the pattern. One sentence. Do not problem-solve in this subflow.
Route to `workflows/personal-drift.md` if a pattern needs addressing.
