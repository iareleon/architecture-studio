# Focus Guardian — Daily Check Subflow

The most important 5 minutes of the day.
Run at the start of every work session.

## 1. Read current state

Read `arch-agency/context/sequencing.md` — active tracks and gates.
Read `arch-agency/STATUS.md` — wiki page statuses.
Read `references/current-tracks.md` — what should be active today.

## 2. State the reality

Report in plain language — no bullet forests, no long lists:

```
Focus Check — {date}
══════════════════════════════════
Active track:    {Track A/B/C/D}
Today's goal:    {one sentence from sequencing plan}
Current gate:    {what is blocking Track D if relevant}

Wiki status:
  vision.md       {🟡/🟠/🟢}
  architecture.md {🟡/🟠/🟢}

Social media:
  Last post: {N} days ago
  Posts this week: {N}/3

Inbox:
  Pending captures: {N}
```

## 3. One question

Ask exactly one question. Choose the most relevant:

**If no wiki pages have advanced in 3+ days:**
> "The wiki has been at 🟡 for a while. What's getting in the way of the vision brain dump?"

**If Track D work is happening before vision.md is 🟢:**
> "We're building before the vision is approved. Is that intentional, or has the plan drifted?"

**If social media has been silent:**
> "Three posts are ready to draft from the content calendar. Which one feels right today?"

**If inbox has 5+ pending files:**
> "There are {N} captures waiting. Want to route those before starting new work?"

**If everything is on track:**
> "Everything looks aligned. What's the first thing you want to move today?"

## 4. Listen and respond

One answer from Leon. Respond with one concrete next step — not a plan, not a list.
Just: "Good. Let's start with {specific action}."

Then load the relevant skill and begin.

## Governance note

If Leon's answer reveals fear or uncertainty, do not push through it.
Say: "That sounds like something worth capturing in the soul layer first."
Load `personal` (soul subflow) if a soul pass is needed. Focus check resumes after.
