# Wiki Sync

Generates or updates `_super-wiki.md` and per-vault pages. Uses a log-diff protocol — only vaults with changes since the last sync are re-read.

## Step 0 — Read last sync timestamp

Check if `{obsidian_root}/_super-wiki.md` exists.

- If it exists: extract the `generated:` value from YAML frontmatter. Store as `last_sync`. Also parse the existing **Vault Registry** table and store rows keyed by vault name for reuse.
- If it does not exist: set `last_sync = null` (forces full read of all vaults).

## Step 1 — Discover vaults

Read vault list from `{OBSIDIAN_META}/wiki-manager.config.yaml` (all entries regardless of `inbox` flag).

As a fallback (if config is absent): scan `{obsidian_root}/` for immediate subdirectories containing both `CLAUDE.md` and `_os/index.md`. Exclude: `inbox/`, `planning/`, `meta/`, hidden directories. Sort alphabetically.

## Step 2 — Read vault metadata (log-diff)

For each vault:

**2a — Change check**

Read `{vault}/_os/log.md`. Extract the date from the most recent `## [YYYY-MM-DD]` heading.

- If `last_sync` is null OR the most recent log date > `last_sync`: vault has changed → proceed to **2b**.
- Otherwise: vault is unchanged → reuse existing row from Step 0. Skip to next vault.

**2b — Full metadata read (changed vaults only)**

| Field | Source | How to extract |
|---|---|---|
| `purpose` | `{vault}/CLAUDE.md` | First non-heading paragraph or line after `# Purpose` / `## Mission` |
| `status` | `{vault}/wiki/` file count | `active` if >2 wiki pages; `scaffold` otherwise |
| `raw_count` | `{vault}/raw/` | Count `.md` files (exclude `_structure.md`) |
| `research_count` | `{vault}/research/active/` + `research/pending-review/` | Count `.md` files total |
| `approved_count` | `{vault}/approved/` | Count `.md` files |
| `wiki_count` | `{vault}/wiki/` | Count `.md` files recursively (exclude `index.md`, `log.md`) |
| `last_op` | `{vault}/_os/log.md` | Date from the most recent `## [YYYY-MM-DD]` entry |

If a file or folder does not exist: use `—` for string fields and `0` for counts.

## Step 3 — Read inbox state

Scan `{obsidian_meta}/inbox/`:
- Count `.md` files in `raw/`, `review/`, and `approved/` subfolders
- Record most recent timestamp from `.classify.log` if present (last line containing a date)

## Step 4 — Write `_super-wiki.md`

Write to `{obsidian_root}/_super-wiki.md`. Always overwrite entirely — do not partially merge.

Output format:

```markdown
---
generated: {YYYY-MM-DD}
skill: wiki-manager
---

# Knowledge OS — Super Wiki

> Auto-generated by wiki-manager. Read this file to orient in the vault.
> Re-run wiki-sync to refresh.
> Per-vault detail: `{obsidian_meta}/wiki/{vault}.md`

---

## Vault Registry

| Vault | Purpose | Status | Raw | Research | Approved | Wiki | Last Op | Detail |
|---|---|---|---|---|---|---|---|---|
| {vault} | {purpose} | {active\|scaffold} | {N} | {N} | {N} | {N} | {YYYY-MM-DD} | [→]({obsidian_meta}/wiki/{vault}.md) |

---

## Inbox State

- **Pending review:** {N}
- **Pending dispatch:** {N}
- **Last classified:** {YYYY-MM-DD or —}
```

Purpose field: max 80 characters, truncate with `…`. Full text in vault page.

## Step 4b — Write per-vault pages (changed vaults only)

For each vault re-read in Step 2b, create or overwrite `{obsidian_meta}/wiki/{vault}.md`.

Output format:

```markdown
---
vault: {vault}
---

# {vault}

{purpose — full text, not truncated}

**Current focus:** {focus — from existing page's `**Current focus:**` line; use `—` if absent}

## Program

{Preserve verbatim if present. Do not overwrite. Omit section entirely if absent.}
```

Create `{obsidian_meta}/wiki/` if it does not exist.

## Step 5 — Folder structure sync for meta

After writing vault pages, run `folder-structure-sync.md` for `{obsidian_meta}/wiki/`.

## Step 6 — Report

```
wiki-sync: {obsidian_root}
  Vaults discovered: {N}  (changed: {N}, unchanged: {N})
  Vault pages written: {N} → {obsidian_meta}/wiki/
  Inbox: {N} pending review, {N} pending dispatch
  → _super-wiki.md updated
```

## Rules

1. Writes only to `{obsidian_root}/_super-wiki.md` and `{obsidian_meta}/wiki/*.md`. No other files modified.
2. Always overwrite `_super-wiki.md` entirely — do not partially merge.
3. Vault pages: only regenerate for vaults re-read in Step 2b. Unchanged vault pages are not touched.
4. Log-diff reuse: a vault row is only reused if its `_os/log.md` date has not advanced past `last_sync`. When in doubt, re-read.
5. Model tier is haiku — aggregate structure only. Do not summarise, interpret, or analyse content.
