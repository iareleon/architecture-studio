Why SkillsLoom

Skills provide procedural knowledge on demand

Agents often lack the context needed to work reliably in your specific environment. Skills close that gap — and do it portably.

For skill authors

Build once, deploy across multiple agent products. Version-control your expertise alongside your code.

For compatible agents

Extend LLM capabilities without retraining. Skills are loaded on-demand, keeping context windows lean.

For teams & enterprises

Capture organisational knowledge in portable, version-controlled packages. Onboard agents to your stack in minutes.

How it works

Filesystem-first. No database required.

Skill state is encoded in directory location, not in a config file. Symlinks control what's visible to the LLM.

📁

Skills are directories

Each skill is a folder containing a SKILL.md file. No build step, no registry, no compile target.

🔗

Symlinks = visibility

Active skills are symlinked into ~/.claude/skills/ or ~/.gemini/skills/. Remove the symlink to hide the skill — the data is never deleted.

⚡

Lifecycle in the path

Each skill stays in skills/<name>/; metadata.status in SKILL.md controls LLM symlinks (run skillmanager audit).
